1. Introducción

El presente documento tiene como propósito describir cómo se va a construir el sistema, tomando como base los requerimientos definidos en el SRS o especificación de requisitos.

La aplicación One Language es un sistema móvil diseñado para facilitar la comunicación entre personas con discapacidad del habla y personas oyentes. Su principal función es traducir la lengua de señas a texto y audio en tiempo real mediante la cámara del dispositivo, utilizando tecnologías de visión por computadora e inteligencia artificial.

Alcance

  • Desarrollar una aplicación móvil con acceso a cámara.
  • Reconocimiento de un conjunto limitado de señas.
  • Traducción automática de las señas reconocidas a texto.
  • Diseñar una interfaz inclusiva y clara fácil de usar.
  • Conversión del texto a audio.

El alcance que se quiere lograr con este proyecto es el desarrollo de una aplicación móvil y un sitio web funcional donde las personas que no tengan ninguna discapacidad logren entender lo que las personas mudas expresan mediante señas.

2. Descripción general del sistema

El sistema es una aplicación móvil orientada a la accesibilidad y la inclusión. Su funcionamiento se basa en capturar los gestos del usuario mediante la cámara del dispositivo y convertirlos automáticamente en texto y audio comprensible.

El sistema está orientado a tres tipos de usuarios:

  • Administrador
  • Persona con discapacidad del habla
  • Persona oyente

El sistema cuenta con cuatro módulos principales:

  • Registro e inicio de sesión: permite a los usuarios crear una cuenta y acceder al sistema.
  • Reconocimiento de señas: detecta y clasifica las señas realizadas por el usuario.
  • Conversión de texto a audio: transforma el texto traducido en audio comprensible.
  • Configuración y accesibilidad: permite personalizar la experiencia del usuario según sus necesidades.

4. Diseño de componentes o módulos

El sistema se organiza en los siguientes módulos funcionales:

1. Módulo de Autenticación

Este módulo permite gestionar el acceso de los usuarios al sistema mediante el registro y validación de credenciales.

  • Entradas: nombre, correo electrónico, contraseña y datos de registro.
  • Procesos: validación de credenciales, registro de nuevos usuarios y recuperación de contraseña.
  • Salidas: acceso al sistema o mensaje de error en caso de datos incorrectos.

2. Módulo de Reconocimiento de Señas

Este módulo se encarga de capturar los gestos realizados por el usuario y convertirlos en texto mediante el uso de reconocimiento de señas.

  • Entradas: video o imagen capturada desde la cámara.
  • Procesos: detección de gestos, análisis del movimiento de las manos y conversión de las señas en texto.
  • Salidas: texto traducido mostrado en la pantalla.

3. Módulo de Conversión a Audio

Este módulo transforma el texto obtenido de la traducción en audio para facilitar la comunicación con personas oyentes.

  • Entradas: texto generado por el módulo de reconocimiento de señas.
  • Procesos: conversión de texto a voz y control de reproducción del audio.
  • Salidas: audio reproducido con la traducción de la seña.

4. Módulo de Historial

Este módulo permite almacenar y consultar las traducciones realizadas dentro de la aplicación.

  • Entradas: traducciones generadas por el sistema.
  • Procesos: almacenamiento de registros, consulta, filtrado por fecha y eliminación de traducciones.
  • Salidas: listado de traducciones guardadas.

5. Módulo de Accesibilidad

Este módulo permite adaptar la interfaz del sistema para mejorar la experiencia de uso según las necesidades del usuario.

  • Entradas: configuraciones seleccionadas por el usuario (tamaño de texto, colores, idioma).
  • Procesos: aplicación y almacenamiento de las preferencias de accesibilidad.
  • Salidas: interfaz personalizada según las configuraciones elegidas.

6. Módulo de Perfil

Este módulo permite al usuario gestionar su información personal y las configuraciones de su cuenta dentro del sistema.

  • Entradas: datos del usuario y configuraciones de la cuenta.
  • Procesos: visualización y actualización del perfil, gestión de permisos y cambio de contraseña.
  • Salidas: perfil actualizado o cierre de sesión del usuario.

8. Reglas de negocio

  • El usuario debe registrarse para utilizar la aplicación.
  • El correo electrónico no puede repetirse.
  • El sistema debe validar credenciales antes de permitir acceso.
  • Las traducciones deben guardarse automáticamente.
  • El sistema debe confirmar antes de eliminar traducciones.
  • Se debe verificar la contraseña actual antes de cambiarla.
  • Si un gesto no es reconocido, el sistema solicitará repetir la seña.

9. Seguridad del sistema

  • Cifrado de contraseñas.
  • Autenticación de usuarios.
  • Bloqueo de cuenta tras 5 intentos fallidos.
  • Protección de datos personales.
  • Gestión de permisos para cámara, micrófono y almacenamiento.

10. Especificaciones técnicas

  • Sistema operativo: Android 10 o superior
  • Frontend: HTML, CSS, JavaScript
  • Backend: Python / Java
  • Base de datos: PostgreSQL
  • Control de versiones: GitHub

11. Consideraciones de implementación

  • Compatibilidad con diferentes dispositivos Android.
  • Optimizar el consumo de batería.
  • Precisión mínima del 85% en reconocimiento de señas.
  • Latencia máxima de 5 segundos.
  • Arquitectura modular para facilitar mantenimiento.
  • Monitoreo del rendimiento del sistema.
